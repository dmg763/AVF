WEEK TWO - DATA RESEARCH, PROJECT INTENT

1. Less obvious perks of using 'live' data resources include logins and payments. How could Twitter be used in these two scenarios?
I believe Twitter can be used the same as Facebook to create or login to different sites and applications. This would grab the users information from their Twitter account such as name, address, etc. This type of login is automatic once created unless the user logs out. The same would be true for payments. Twitter would associate a user with a specific payment service. Most likely this would be PayPal as this service is widely accepted.
2. Visit Programmableweb.com and search for JSON-P data sources. Why would we need JSON-P, specifically?
If I understand correctly, JSON-P is used for cross-domain requests. This allows the user to access data outside of the current application or site.
3. Based on the JSON-P search results: what data resource(s) do you think might pair well with which types of Native device features?
Camera will pair well with many of the APIs. Some of these might include advertising, blogging, entertainment, social networking.
Contacts could pair well with databases, social networking, e-mail, file sharing, and games.
Notification could pair well with e-mail, social networking, entertainment, financial, job search, and messaging.
4. Please describe any design and functionality intentions you may have for your Demo app components this month.
I was hoping to combine many of the native features with an entertainment API. I was thinking the user could enter a zip code to find local entertainment. This may include concerts, pumpkin patches, theatre, amusement parks, restaurants, etc. The user could receive notifications, take pictures, and share with contacts. This may change as the entertainment APIs I am finding so far, are not really what I was thinking about. I will continue my search and see if I can find something more suitable.
5. Briefly describe some past 'lessons learned' regarding workflow as it relates to research, design, development, etc.
One lesson learned that I do entirely too often is to “over think” what is required. When I read the instructions, it sounds so simple, I believe I must be missing something.
For the most part, I do start as early as possible; however, when I run into issues, I tend to feel frustrated and stuck. If I cannot resolve the issue, I try to do more research and reach out to others. It have learned it is in my best interest, while waiting for assistance, to work on something else instead of dwelling on that one specific issue. I feel more productive and as if I have at least accomplished something.
